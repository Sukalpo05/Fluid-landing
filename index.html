Traditional frontend stacks require writing separate layouts or maintaining massive bundles of conditions to customize UI for different users. Fluid ends the code sprawl.

The Three-Step Cycle

How Fluid delivers native-speed, custom layouts on-the-fly without running client-side LLMs.

01

Define Capability

Write a standard schema representing your business actions. Fields, actions, and validations are fully typed in TypeScript and uploaded to Fluid registry.

02

Supply Context & Intent

Send user device specifications, historical interactions, regional data, and explicit requests to our edge resolution nodes.

03

Compile & Render

The compiler evaluates rules, runs layout compilation, outputs a secure JSON-based IR, and your SDK paints the view instantly.

Minimal Integration. Maximum Control.

Look under the hood at the schema declaration, the Intermediate Representation (IR), and client-side rendering.

Typescript 
import { defineCapability } from "@fluid-ui/sdk";

export const feedbackForm = defineCapability({
  id: "user-feedback",
  version: "1.2.0",
  fields: {
    rating: { type: "integer", min: 1, max: 5, required: true },
    message: { type: "string", required: true },
    email: { type: "email", required: false }
  },
  actions: {
    submit: {
      endpoint: "/api/feedback",
      method: "POST"
    }
  }
});
{
  "$schema": "https://fluid.dev/ir/v1.json",
  "id": "user-feedback",
  "layout": {
    "style": "glassmorphic",
    "spacing": "compact"
  },
  "components": [
    {
      "type": "header",
      "props": { "title": "Outage Feedback", "alert": true }
    },
    {
      "type": "stars-selector",
      "binding": "rating",
      "props": { "label": "Rate your outage experience" }
    },
    {
      "type": "textarea",
      "binding": "message",
      "props": { "placeholder": "What went wrong?" }
    },
    {
      "type": "submit-btn",
      "action": "submit",
      "props": { "variant": "danger", "text": "Escalate Feedback" }
    }
  ]
}
import { FluidRenderer, useFluid } from "@fluid-ui/react";

export default function App() {
  // Instantly fetches compiled IR and handles action execution securely
  const { ir, loading, executeAction } = useFluid({
    capability: "user-feedback",
    context: { device: "desktop", isOutageActive: true }
  });

  if (loading) return <div>Loading Fluid interface...</div>;

  return (
    <main className="dark bg-black p-12">
      <FluidRenderer 
        ir={ir} 
        onAction={executeAction} 
      />
    </main>
  );
}

Why an IR Compiler Wins

Generating JavaScript directly from AI is slow, insecure, and breaks on updates. Fluid uses an Intermediate Representation (IR) to ensure speed, security, and predictability.

Deterministic Sandbox

Since the output is a JSON-based schema, it runs within a sandbox with strict capability boundaries. Code injections, broken third-party script runs, and unexpected client crashes are mathematically impossible.

Edge-Cacheable Layouts

Dynamic evaluations take place in Vercel / Cloudflare Edge Workers in under 5ms. The resulting IR payload is tiny (typically less than 2KB), resulting in faster page loads and zero bundle hydration lag.

Cross-Platform Native

You aren't locked to HTML. The same Fluid IR compiles natively into React Components, Swift UI for iOS apps, Jetpack Compose for Android, or Flutter views. Code once, personalize everywhere.
